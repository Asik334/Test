# NumericInput — Техническое задание

Production-grade числовой инпут с живым форматированием, адаптивной шириной и аккуратным сохранением позиции курсора — React + TypeScript, без сторонних библиотек.

---

| Инструмент | Версия | Зачем |
|---|---|---|

## Что реализовано

### Переиспользуемый `NumericInput`

Полноценный контролируемый/неконтролируемый компонент с `forwardRef`:

```tsx
// Controlled
<NumericInput value={rawDigits} onChange={setRawDigits} />

// Uncontrolled
<NumericInput defaultValue="42" />
```

Принимает `value` как чистые цифры (`"1000000"`), отображает форматированный вид (`"1 000 000"`).

---

### Форматирование

Числа разделяются **узким неразрывным пробелом** (U+202F) — именно его используют европейские стайлгайды как разделитель тысяч. Это исключает путаницу с обычным пробелом, который некоторые парсеры читают как десятичный разделитель.

```ts
formatNumber('1000000')  // '1 000 000'
formatNumber('42')       // '42'
stripFormatting('1 000 000') // '1000000'
```

---

### Адаптивная ширина через Canvas

Ширина пересчитывается после каждого рендера через **offscreen Canvas** (один синглтон на весь модуль):

```ts
ctx.font = `${weight} ${size} ${family}`
const measured = ctx.measureText(displayValue || placeholder).width
width = Math.max(minWidth, measured + paddingH + borderH + 4)
```

`+4` — курсор (2px) + breathing room, чтобы значение не уезжало внутри инпута. Это честное измерение текста, а не `value.length * 20`.

---

### Сохранение курсора при форматировании

Переформатирование меняет длину строки — пробелы добавляются и убираются. `remapCursor` считает, сколько **цифр** стоит левее курсора в старой строке, потом находит ту же позицию в новой:

```
old:  "1 000|"   → 4 цифры левее курсора
new:  "10 000"
       ↑ идём до 4-й цифры → позиция 5 → "10 00|0"
```

---

### Paste с автоформатированием

Вставка `"1000000"` → автоматически `"1 000 000"`. Вставка в середину числа тоже работает корректно: не-цифры вырезаются, цифры вставляются в позицию курсора.

---

### Dark Mode

Переключение темы через `ThemeContext` + токены дизайна. Каждый токен — это пара значений `light/dark`:

```ts
const tokens = {
  light: { bg: '...', card: '...', inputBg: '#fff', ... },
  dark:  { bg: '...', card: '...', inputBg: 'rgba(255,255,255,0.05)', ... },
}
```

Все переходы сглажены через `transition: ... 200ms` на каждом элементе. Блобы, карточка, инпуты, стрелки — всё меняет цвет одновременно.

---

### Reset

Кнопка сбрасывает все три значения к дефолтным через один `useCallback`. Никакого "Submit" или "Save" — только то, что реально полезно.

## Обработка edge-cases

| Кейс | Поведение |
|---|---|
| Ведущие нули | Убираются (`007` → `7`) |
| Вставка с не-цифрами | Не-цифры вырезаются, цифры вставляются в позицию курсора |
| Пустое значение | Показывается placeholder, ширина = minWidth |
| Очень большие числа | JS `Number` держит до 2⁵³; отображение остаётся корректным |
| Delete посередине | Цифра удаляется, пробелы перераспределяются, курсор ремаппится |
| Mobile paste | `onPaste` перехватывает буфер до того, как ОС успевает его отформатировать |

## Доступность

- `inputMode="numeric"` — числовая клавиатура на iOS/Android
- `role="spinbutton"` + `aria-valuemin={0}` — корректная ARIA-семантика
- `aria-label` пробрасывается через props (дефолт: `"Numeric input"`)
- `font-size: max(16px, …)` — предотвращает zoom Safari при фокусе
- `autoComplete="off"`, `autoCorrect="off"` — чистый мобильный опыт
- `@media (prefers-reduced-motion)` — отключает анимацию блобов

---

## Responsive

- Карточка: `max-width: calc(100vw - 32px)` + `width: fit-content`
- `overflow-x: hidden` на `body` — нет горизонтального скролла
- Блобы на GPU: `will-change: transform`
- Отступы меняются на `<400px` через медиа-запрос

## Загрузка изображения

```ts
const CAT_IMG = new URL('./img.png', import.meta.url).href
```

Vite хэширует и инлайнит файл как base64 (порог < 50KB в `vite.config.ts`). Работает на любом хосте без `public/` и без проблем с путями. `onError` скрывает `<img>` если ассет недоступен.
